⚡ Análisis de Código en Tiempo Real

Módulo: Programación C

❌ Lo que respondiste:

"En mi función para asignar memoria a un puntero, utilizo free(ptr); ptr = NULL; y luego intento acceder a *ptr para mostrar el valor que tenía, así no hay fugas de memoria."

✅ Corrección de la IA:

Tu lógica para evitar fugas de memoria es correcta al usar `free(ptr)`, pero tu respuesta tiene un error crítico de lógica de programación.

Al hacer `ptr = NULL` después de liberarlo, evitas un puntero colgante ("dangling pointer"), lo cual es una mejor práctica. Sin embargo, intentar acceder a `*ptr` (desreferenciar) inmediatamente después de asignarle `NULL` provocará un error de segmentación (segmentation fault) y el programa se cerrará.

Respuesta sugerida: "Una vez liberada la memoria con `free(ptr)` y asignado `NULL`, no es posible acceder a su contenido. Lo correcto es mostrar el valor que tenía *antes* de liberarlo."
